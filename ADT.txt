Aquí tienes el paso a paso de qué pedirle a Antigravity (en modo PLANNING) para cada caso:

Caso 1: Diseño o Responsive
El riesgo: Que la IA te cambie el CSS y rompa la estética general o use librerías que no querés.

Qué pedir: "Quiero que la interfaz [X] sea responsive. Revisá el SOP de Arquitectura para ver qué framework de diseño estamos usando. No toques el código hasta que confirmes que el cambio no rompe las reglas estéticas del SOP."

Resultado: Antigravity verifica si hay reglas de "Mobile-First" en tus SOPs y aplica el cambio de forma coherente.

Caso 2: Finanzas no refleja bien la información (Error de Lógica)
El riesgo: Que la IA "parchee" el código pero el error de cálculo siga ahí.

Qué pedir: "La sección de finanzas muestra datos erróneos. 1. Buscá en el SOP de Análisis cómo se supone que se calcula el [Total/Impuesto/etc]. 2. Compará esa lógica con el código actual. 3. Si el SOP está bien pero el código mal, arreglá el código. Si el SOP es el que está incompleto, actualizalo primero."

Resultado: Asegurás que el cálculo matemático quede documentado para siempre y no vuelva a fallar.

Caso 3: Nueva sección (Tabla con Indicadores)
El riesgo: Que invente nombres de tablas o use indicadores que no existen en la base de datos.

Qué pedir: "Quiero agregar una tabla de indicadores en el módulo [X]. 1. Leé los SOPs maestros. 2. Actualizá el SOP de Arquitectura con los nuevos endpoints y la estructura de la tabla. 3. Actualizá el SOP de Análisis definiendo qué significa cada indicador. 4. Una vez guardados los .md, programá la sección."

Resultado: El sistema crece de forma ordenada. Si mañana otro programador (o vos mismo) quiere ver de dónde salen esos indicadores, están en el SOP.

Caso 4: Error en Consola (Bugs técnicos)
El riesgo: Que solucione el error pero ignore las reglas de seguridad o de arquitectura del proyecto.

Qué pedir: "Tengo este error en consola: [Pegar error]. 1. Analizá por qué sucede. 2. Antes de arreglarlo, verificá en el SOP de Arquitectura el protocolo de manejo de errores y validaciones. 3. Corregilo asegurando que no viole ninguna 'Regla de Oro' del SOP de Análisis."

Resultado: El error se va, pero la robustez del sistema se mantiene intacta.

Regla de oro para los 4 casos:
Cuando Antigravity te diga: "Ya está listo", tu respuesta final siempre debe ser:

"¿Actualizaste los archivos .md en /docs para reflejar estos cambios?"

Si dice que no, pedile que lo haga. El SOP desactualizado es el inicio del caos.
